# Contributing to Exponential Platform

Thank you for considering contributing to Exponential Platform! We appreciate the time and effort you invest in making this project better.

## Licensing

Exponential Platform is licensed under the **GPLv2 or later** license. Any code contributions must be released under the same license. This ensures that improvements remain open-source and available to the entire community.

## Code of Conduct

All contributors are expected to be respectful and constructive. Harassment of any kind will not be tolerated.

## Getting Started

1. Fork the repository and clone your fork:

```bash
git clone https://github.com/your-username/<repository>.git
```

2. Install project dependencies:

```bash
composer install
```

3. Create a feature branch:

```bash
git checkout -b feature/your-feature
```

## Making Changes

- Always work on a named branch — never commit directly to `master` or `main`.
- Follow [PSR coding standards](https://www.php-fig.org/psr/).
- Write clear commit messages following [Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/).
- Keep pull requests focused. Submit separate PRs for separate concerns.

## Submitting a Pull Request

1. Ensure tests pass.
2. Update documentation if your change affects behaviour.
3. Push your branch and open a pull request against `master`.

```bash
git push origin feature/your-feature
```

All contributions are reviewed before merging. Be open to feedback — code review is how we maintain quality.
